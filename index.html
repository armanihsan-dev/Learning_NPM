NPM (Node Package Manager) Detailed Notes

Introduction

NPM stands for Node Package Manager. It is a tool used to manage JavaScript packages and dependencies for Node.js projects. It is the default package manager for Node.js and comes bundled with it.

Advantages: Why We Use NPM?

  • Dependency Management: Allows you to easily manage and update dependencies for your project.
  • Removes Global Space Conflicts: By installing packages locally, it prevents global namespace pollution and version conflicts.
  • Access to Open-Source Libraries: Provides access to a vast library of reusable JavaScript code.
  • Automates Tasks: Simplifies tasks like bundling, minifying, testing, and more.

How NPM Works

NPM works by maintaining a registry of packages (also known as modules). These packages can be downloaded and installed into your project via the command line. It also manages a file called package.json, which keeps track of the dependencies and scripts for your project.

Check NPM Version

To check if NPM is installed and see its version, use the following command in your terminal: 

npm -v

This will display the currently installed version of NPM.

What is node_modules?

The node_modules folder is where NPM installs all the dependencies for your project. It contains the actual source code of the libraries you install, as well as their sub-dependencies. This folder should generally not be modified manually.

Tip: The node_modules folder is typically ignored in version control using a .gitignore file.

Setting "type": "module" in package.json

To use ES modules (import/export syntax) in your project, you need to add the following line in your package.json file: 

{
  "type": "module"
}

This allows you to use modern JavaScript syntax like import and export.

Bundlers and Parcel

A bundler is a tool that combines multiple JavaScript files, along with other assets like CSS and images, into a single file. This improves performance by reducing the number of HTTP requests.

  • Popular Bundlers: Webpack, Parcel, Vite, and Rollup.
  • Why Parcel? Parcel is easy to set up, fast, and supports zero-configuration bundling.

Setting Up Parcel

To use Parcel in your project, follow these steps:

  1. Install Parcel as a dev dependency:
    2. npm i parcel -D
  2. Create an index.html file and include your JavaScript file: 
    3. <script src="script.js" type="module"></script>

Starting a Parcel Server

Once Parcel is set up, you can start a local development server with the following command: 

npx parcel index.html

This will host your application on http://localhost:1234, enabling live reloading for changes.
